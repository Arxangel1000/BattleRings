Игра представляет собой поле с юнитами, которые сражаются друг с другом за место в песочнице. 

Реализованный функционал: 
Считывание и обработка данных из json файла.
Загрузка игрового поля.
Проверка возможных позиций для спавна юнитов.
Спавн синих юнитов.
Спавн красных юнитов.
Рандомное распределение характеристик юнитов.
Запуск игровой баталии.
Сама баталия.
Табличка победы с данными.
Какая команда победила.
Время игровой сессии.

Интерфейс:
У игрока есть возможность регулировать скорость битвы при помощи бара.
У игрока есть возможность при помощи свайпа покрутить камеру вокруг игрового поля.
Игрок может перевернуть телефон в горизонтальное положение
Есть кнопка рестарт игровой сцены.
Есть двухцветный бар со статистикой баланса юнитов 

РАЗВЕРНУТЫЙ ОТЧЕТ:
Основные настройки игры хранятся в скрипте GameData, который получает данные из нашего json файла.
В своем проекте я применил разные элементы построения архитектуры. 
Где-то я воспользовался контроллерами, где-то применил глобальные статические классы для хранения данных. Юниты были реализованы с применением MVC. События и перечисления вынес в отдельные классы.
Так же я использовал несколько патернов, вроде синглтона, слушателя и фабрику для создания игровых объектов.
Собственно, вся архитектура.
Во время выполнения задания возникла сложность со спавном юнитов, а точнее с их перекрытием друг друга. Я мог бы применить сетку, но решил пойти другим путем, ведь при сетке возможно, что в некоторых вариантах юниты располагались бы слишком очевидно стройно. 
Мне хотелось, чтобы они появлялись на игровом поле более естественно. Поэтому пришлось повозиться с механизмом исключения занятых позиций. Перед созданием юнитов, я использовал систему антиоверлапинга для проверки свободных мест на поле. Поэтому юниты располагаются четко и рандомно. 
Для некоторых вещей я задействовал систему событий. Например, для оповещения юнитов о конце построения и начале движения.
При создании бара мне хотелось реализовать идею борьбы юнитов, поэтому у него есть две характерные полосы, подобные юниты на поле в процентном соотношении. Бар отображает баланс общего здоровья (радиусов) юнитов.
Так же я реализовал мультиоконность. Играть можно как в вертикальном, так и в горизонтальном положении. Игра динамически определяет, положение экрана.
Касаемо физики юнитов попробовал несколько решений, но в последствии остановился на варинате с transform.Translate и сменой угла движения при столкновении. Для границ поля сперва попробовал луч, но потом решил поставить простые стенки из колайдеров.
Я не стал делать анимацию кружков, по причине того, что не хотелось загромождать процесс баталии анимацией. Но с анимацией у меня все в порядке в плане навыков. Я раньше много работал с меканимом.
Так же сделал возможность крутить камеру вокруг поля при помощи свайпа, если провести пальцем по любой области вне UI. Чтобы при перемещении ползунка бара скорости камера не крутилась я поставил запрет на свайп при касании UI.
Основную часть ТЗ я выполнил. Камера свайпается, экран переворачивается, спавн и игровая сессия реализованы, в конце есть табличка со статистикой.
